# lc-review —— 间隔复习（3→7→15 递增）

对到期的旧题做主动回忆式复习。核心原则：**先不给答案**，逼用户凭记忆重建思路（主动回忆比重读笔记记得牢得多），再对照笔记打分，按表现更新复习间隔。

## 数据位置

数据目录由环境变量 `LC_DATA_DIR` 决定，未设置则默认 `$HOME/leetcode`。
先用 Bash 解析：`DIR="${LC_DATA_DIR:-$HOME/leetcode}"`。

- 复习表：`$DIR/_复习表.md`
- 题解笔记目录：`$DIR/题解/`

## 执行步骤

### 第一步：找出到期的题

1. Bash 取今天日期 `date +%Y-%m-%d`。
2. 读复习表，筛出「待复习 / 进行中」里 `下次复习 <= 今天` 的题。
3. 没有到期的 → 告诉他今天不用复习，可以去 `/lc-go` 刷新题。
4. 有多道 → 一次复习一道，从最久没碰（下次复习日期最早）的开始。

### 第二步：主动回忆（先别给答案！）

报出题号题名，**不要打开笔记给他看**，让他凭记忆回答：
- 这题用什么思路/数据结构？为什么？
- 关键步骤是什么？复杂度多少？
- 让他口述，或直接在编辑器里**默写一遍代码**。

### 第三步：对照笔记考核

读这题的题解笔记，对照他的回答：
- 答对/答全 → 点出他记得牢的地方。
- 漏了或答错 → 指出差距，重点讲他忘掉的"关键卡点"和"一句话记忆点"。
- 简洁点评，不啰嗦。

### 第四步：判定掌握程度，更新复习表

让他（或你根据表现）判定这次复习结果：

- **顺利答出（通过）**：阶段 +1，按新阶段算下次复习日期，掌握度升一级。
  - 阶段 0→1：`date -v+7d +%Y-%m-%d`（+7 天），掌握度 🌿
  - 阶段 1→2：`date -v+15d +%Y-%m-%d`（+15 天），掌握度 🌳
  - 阶段 2→3：**毕业** 🎓 —— 从「待复习」表删除，移到「已毕业」表，毕业日期=今天。
- **没答好（没通过）**：阶段重置为 0，下次复习 = `date -v+3d +%Y-%m-%d`（+3 天），掌握度降回 🌱。

（非 macOS 用 `date -d "+7 days" +%Y-%m-%d` 形式。）

更新对应行的「阶段 / 下次复习 / 掌握度」，并把「复习次数」+1。

> [!note] streak
> 复习也算当天有效练习。按 lc-done 里相同的 streak 规则更新 frontmatter 的 `streak` / `last_active`（last_active==今天则不变；==昨天则 +1；否则置 1）。

### 第五步：继续或收尾

如果还有到期的题，问要不要复习下一道。全部复习完 → 一句话总结今天复习了几题、几题毕业，鼓励一句。
