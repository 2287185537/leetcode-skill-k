# lc-done —— 做完复盘（逼你说明白 + 整理）

用户做完一题。核心原则：**不要直接替他写笔记**。先用提问逼他把思路讲明白（讲不明白 = 没真懂），用他的回答 + 代码，生成简洁清晰的题解笔记，并登记 3 天后复习。

## 数据位置

数据目录由环境变量 `LC_DATA_DIR` 决定，未设置则默认 `$HOME/leetcode`。
先用 Bash 解析：`DIR="${LC_DATA_DIR:-$HOME/leetcode}"`。

- 题单：`$DIR/_题单.md`
- 复习表：`$DIR/_复习表.md`
- 题解笔记目录：`$DIR/题解/`（不存在就创建；文件名 `NNNN-题名.md`，如 `0046-全排列.md`）

## 执行步骤

### 第一步：拿到题和代码

确认是哪道题、让用户贴出他 AC 的代码（如果还没贴）。

### 第二步：苏格拉底式追问（这是重点，不能跳过）

**先别写笔记**，针对这道题问 3-4 个直击理解的问题，一次性抛出，让他回答。问题要具体到这道题，不要泛泛。常用角度：

- **为什么是这个解法/数据结构**：为什么用哈希表而不是排序？为什么这里要用单调栈？
- **卡点**：你卡在哪一步最久？哪个地方想了很久才通？
- **边界 / 易错**：哪些边界条件容易漏（空输入、单元素、重复值、越界）？
- **复杂度**：时间空间是多少，瓶颈在哪？
- **DP/回溯类**：状态怎么定义的？转移方程为什么这样写？递归边界是什么？

如果他答得含糊或答错，**追问或点破**，确保真懂了再往下。这一步的价值就在于他被迫输出。

### 第三步：生成笔记

用他的回答 + 代码，写入题解目录。风格 = **简洁清晰、从暴力到最优的思路推导、明确状态定义、清晰复杂度、可复用模板**（参考灵茶山艾府的题解风格）+ Obsidian 排版（callout / frontmatter）。模板：

```markdown
---
tags:
  - leetcode
  - <专题，如 回溯>
题号: <N>
难度: <简单/中等/困难>
专题: <专题>
created: <今天日期>
aliases:
  - <英文题名>
---

# N. 题名

> [!abstract] 一句话本质
> **<这道题的核心判断>**，不是……而是……

> [!quote] 链接
> - https://leetcode.cn/problems/<slug>/

## 题目
<一两句话简述，给未来的自己看>

## 思路（从暴力到最优）
<讲清"怎么想到的"：先最直接的想法 → 为什么慢 → 怎么优化。
DP 题必须写清：状态定义 dp[i] 表示什么、转移方程、边界、答案在哪。>

## 关键卡点
> [!warning] 我当时卡在这
> - <来自第二步问答的真实卡点，用他自己的话>
> - <易错边界>

## 代码
```python
<他 AC 的代码，必要时清理变量名>
```

## 复杂度
- 时间：O(...)
- 空间：O(...)

## 一句话记忆点
> [!tip] 下次看到类似题
> <一句能触发回忆的钩子，比如"找两数之和→边遍历边查哈希表"。>

## 相似题 / 套用模板
- <同专题可迁移的题或模板>
```

写完告诉用户笔记路径。

### 第四步：登记进度（改两个文件）

1. **题单**：把这道题的 `- [ ]` 改成 `- [x]`。
2. **复习表**：
   - 用 Bash 算下次复习日期：`date -v+3d +%Y-%m-%d`（做完后 3 天；非 macOS 用 `date -d "+3 days" +%Y-%m-%d`）。
   - 在「待复习 / 进行中」表格加一行：题号 / 题名 / 专题 / 笔记链接 / 阶段=0 / 下次复习=+3天 / 掌握度=🌱 / 复习次数=0。
   - 更新 frontmatter：`total_done` +1；按下面规则更新 `streak` 和 `last_active`。

**streak 规则**（用今天日期和 frontmatter 里的 `last_active` 比较）：
- `last_active` == 今天：streak 不变（今天已经记过）。
- `last_active` == 昨天（`date -v-1d +%Y-%m-%d`）：streak +1。
- 其它情况（断了或首次）：streak = 1。
- 最后把 `last_active` 设为今天。

### 第五步：收尾

一句话告诉他：✅ 已记录，N 天连续打卡，这题 3 天后（具体日期）会进复习。鼓励一句，问要不要继续下一题（`/lc-go`）。
