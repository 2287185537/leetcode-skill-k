# lc-go —— 打开就开始（治拖延）

用户坐到电脑前但容易拖延不开始。这个 skill 的唯一目标：**用一条命令消除"今天刷什么"的决策成本**，给出明确的"现在就做这道"。

## 数据位置

数据目录由环境变量 `LC_DATA_DIR` 决定，未设置则默认 `$HOME/leetcode`。
先用 Bash 解析：`DIR="${LC_DATA_DIR:-$HOME/leetcode}"; echo "$DIR"`。

- 题单：`$DIR/_题单.md`
- 复习表：`$DIR/_复习表.md`

如果 `_题单.md` / `_复习表.md` 不存在，提示用户先运行仓库里的 `install.sh` 初始化数据目录。

## 执行步骤

1. 用 Bash 取今天日期 `date +%Y-%m-%d`（不要用记忆里的日期）和数据目录。
2. 读复习表「待复习 / 进行中」表格，筛出 `下次复习 <= 今天` 的题 = **今日到期复习**。同时读 frontmatter 的 `streak` / `total_done`。
3. 读题单，从上往下找**第一道未打勾 `- [ ]`** 的题 = **今日新题**。记下它属于哪个专题、专题内进度（如 哈希 2/3）。
4. **不修改任何文件**（lc-go 只读，可重复运行）。
5. 按下面格式输出，简洁、一针见血，给一句轻量激励。

## 输出格式

```
🔥 连续打卡 N 天 ｜ 已完成 X/100

📺 今日复习（M 题到期）
  - 49. 字母异位词分组 ｜ 阶段0 ｜ 🌱  → 待会儿用 /lc-review
  （没有到期的就写：今天没有要复习的，专心攻新题 👍）

🎯 今日新题 → 现在就做这道
  46. 全排列 ｜ 专题：回溯（4/8）
  https://leetcode.cn/problems/permutations/

做完回来敲 /lc-done 复盘。先做新题还是先复习都行，动起来最重要。
```

## 规则

- 链接用 leetcode.cn 标准格式：`https://leetcode.cn/problems/<英文 slug>/`，slug 根据题目推断（如 两数之和 = two-sum）。
- 如果到期复习超过 3 题，提示"今天复习有点多，可以只攻最久没碰的 2-3 道"。
- 如果题单全部打勾完了，恭喜并建议进入纯复习巩固阶段。
- 不要长篇大论，不要讲题，这一步只负责"让他开始"。
